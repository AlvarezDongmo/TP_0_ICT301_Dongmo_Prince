### 0. 
Les principes SOLID représentent cinq règles fondamentales de conception orientée objet qui, lorsqu'elles sont appliquées ensemble, permettent de créer des systèmes logiciels robustes, facile a la maintenance et a l'extensibilite. Chaque principe adresse un aspect spécifique du design : le SRP garantit la cohésion des classes, l'OCP favorise l'extensibilité sans modification, le LSP assure la substituabilité des types, l'ISP élimine les dépendances inutiles, et le DIP inverse les dépendances vers des abstractions. Collectivement, ils réduisent le couplage, augmentent la cohésion, et facilitent l'évolution du code en permettant aux développeurs d'ajouter de nouvelles fonctionnalités avec un impact minimal sur le code existant.

### 1. SRP (Single Responsibility Principle)
Chaque classe doit avoir une seule responsabilité. Une classe ne doit changer que pour une seule raison.

### 2. OCP (Open/Closed Principle)
Les entités logicielles doivent être ouvertes à l'extension mais fermées à la modification.

### 3. LSP (Liskov Substitution Principle)
Les objets d'une classe dérivée doivent pouvoir remplacer les objets de la classe de base sans affecter le fonctionnement du programme.

### 4. ISP (Interface Segregation Principle)
Il vaut mieux avoir plusieurs interfaces spécifiques qu'une seule interface générale.

### 5. DIP (Dependency Inversion Principle)
Il faut dépendre des abstractions, pas des implémentations concrètes.
